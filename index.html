elixir by glennblock

An api to ease some of the pain related to databinindg

Elixir is designed to be a cure for some of the pains associated with databinding in Silverlight and WPF. Primarily this revolves around lack of compile-time checking for bindings, brittleness of bindings as it relates to refactoring (you rename a member and the binding is broken), debugability of bindings. Elixir provides several tools. 1. A fluent api for binding UI elements to models. The API is convention based thus removing a lot of the DRY code. 2. A XAML based convention based API. 3. Pluggable conventions thus allowing you to define your own rules of the road.
